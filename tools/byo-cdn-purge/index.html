BYO CDN Purge

POST your BYO CDN credentials and optional purge targets to the Adobe byocdn-push-invalidation run-anywhere service (helix-pages.anywhere.run/helix-services/byocdn-push-invalidation/v1), the same endpoint used by CDN Check and CDN Setup. Akamai is supported first; other vendors can be added later.

Security: this page can mirror your inputs into the browser URL (query string) for sharing or bookmarking. Query strings are stored in history, visible to extensions, and may leak via the Referer header. Prefer pasting credentials per session and use Remove secrets from URL when you are done sharing the link shape (host / targets only).
Provider

Additional providers will appear here as support is added.

Akamai credentials

Hostname served by your production property (no scheme).

Purge targets (optional)

One entry per line. Leave both empty to run the service default credential check (same idea as CDN Setup validate). When you provide targets, they are sent as repeated urls and keys form fields alongside credentials.

The current values are long; some browsers limit bookmark URL length. Consider shorter lists or run without putting everything in the query string.

Submit the form to see JSON from the push invalidation service.
